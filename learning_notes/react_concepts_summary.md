# React 學習筆記與對話紀錄整理

## 1. 專案檔案結構導覽
對於初學者，此專案的三個靈魂人物：

### `index.html` (房子的地基)
*   **角色**：網頁的入口。
*   **重點**：有一個 `<div id="root"></div>` 空房間。
*   **功能**：所有的 React 程式碼最後都會被塞進這裡。

### `index.tsx` (管家)
*   **角色**：連接橋樑。
*   **功能**：負責把 `App.tsx` (您的程式) 掛載到 `index.html` 的 root 裡面。

### `App.tsx` (主角/樣品屋)
*   **角色**：網頁本體。
*   **功能**：決定畫面長什麼樣子、包含哪些頁面、以及如何切換。
*   **建議**：從這裡開始修改與學習。

---

## 2. 關於 HTML 與 CSS
在 React + Tailwind 的架構下：
*   **HTML (結構)**：`<` 開頭 `>` 結尾的標籤 (如 `<div>`, `<h2>`, `<button>`)。
*   **CSS (樣式)**：寫在 `className="..."` 裡面的那一長串文字 (如 `flex`, `bg-white`, `text-red-500`)。
*   **差異**：不像傳統分開兩個檔案寫，現在是「樣式直接寫在結構上」。

---

## 3. 檔案副檔名解密
### `.ts` (TypeScript)
*   **功能**：只負責**純邏輯**、**資料設定**、**規則定義**。
*   **限制**：**不能**寫 HTML 標籤。
*   **用途**：如 `constants.ts` (資料庫), `types.ts` (規格書)。

### `.tsx` (TypeScript + XML)
*   **功能**：負責**畫出畫面**，可以寫 JSX (HTML 標籤)。
*   **用途**：如 `App.tsx`, `Navbar.tsx` (所有看得到的元件)。

---

## 4. React vs Vue 架構比較
| 特性 | React (.tsx) | Vue (.vue) |
| :--- | :--- | :--- |
| **結構** | **JSX**：邏輯與畫面混寫 (All-in-JS)。 | **SFC**：HTML/JS/CSS 三區塊分離。 |
| **風格** | 像在寫程式 (靈活)。 | 像在寫網頁 (結構清晰)。 |
| **上手度** | 較陡峭，需懂 JS 觀念。 | 較平緩，適合新手。 |
| **優勢** | 生態系大、適合大型應用、React Native。 | 開發快、規範統一、好維護。 |

## 5. 核心 JavaScript/React 語法概念 (生活化比喻)

### Arrow Functions (箭頭函式) `() => {}`
*   **比喻**：**遙控器的紅外線連接**。
*   **意義**：綁定「動作」與「結果」。例如：`onClick={() => 回首頁()}` (手點下去 $\rightarrow$ 執行回首頁)。

### Destructuring (解構賦值) `const { name } = props`
*   **比喻**：**從工具箱只拿出要用的工具**。
*   **意義**：不把整個包裹 (`props`) 拿來用，而是直接把需要的屬性 (`currentPage`, `onNavigate`) 拆出來放在桌上，讓程式碼更乾淨。

### JSX (JavaScript XML) `<div>...</div>`
*   **比喻**：**所見即所得的設計圖**。
*   **意義**：在程式碼裡直接寫 HTML 標籤，寫什麼畫面上就長什麼，不需要在大腦中轉換數學公式。
